Slice 1: Flight Model + hardcoded flight card

Blocked by: None Stories: 1, 2, 3, 4, 11
Core data types (Flight dataclass, FlightBuffer ring buffer) and a renderer that draws flight cards using Departure Mono. A hardcoded Flight makes the end-to-end path demoable: model → render → display.

Acceptance criteria

  • Flight dataclass with display helpers
  • FlightBuffer ring buffer with callsign dedup
  • Departure Mono rendered via Pillow on 64×32 canvas
  • uv run jetset shows a fake flight card on the emulator
  • lint + model tests pass
